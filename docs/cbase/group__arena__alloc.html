◆ ARENA_COMMIT_SIZE

#define ARENA_COMMIT_SIZE   KB(64)

The granularity at which physical RAM is committed from the OS (64 KB).

When an allocation would exceed the currently committed range, the arena rounds the new commit size up to this boundary. Matching the Windows allocation granularity avoids wasted syscalls.

◆ ArenaTemp

typedef struct ArenaTemp ArenaTemp

A saved checkpoint into an arena, used for scoped temporary allocations.

Call arena_temp_begin() to capture the current pos, do your allocations, then call arena_temp_end() to roll the arena back to that exact position. This is the preferred pattern for scratch/temporary work buffers.

Note
Only pos is saved and restored, not cmt. Physical RAM committed during the temp scope stays mapped after arena_temp_end() — it will be reused by future allocations without an extra commit syscall. If you need to actually return those pages to the OS, call arena_clear_and_decommit() instead.
Example
ArenaTemp tmp = arena_temp_begin(&arena);
u8 *scratch = PUSH_ARRAY(&arena, u8, KB(4));
// ... build something temporary in scratch ...
arena_temp_end(tmp); // all temporary allocations are gone

void arena_clear ( Arena * arena)

Resets the arena to empty by setting pos back to 0.

All previously returned pointers are invalidated. Physical RAM remains committed — the next allocations are as fast as before. This is the idiomatic per-frame reset.

◆ arena_clear_and_decommit()

void arena_clear_and_decommit ( Arena * arena)

Resets the arena AND returns all committed physical RAM to the OS.

Use this instead of arena_clear() when the arena held a large one-off allocation and you want the physical pages back (e.g., a level-load scratch arena that is rarely refilled). Subsequent allocations will re-commit RAM on demand as normal.

Note
The virtual address reservation is kept; only physical RAM is released.
Parameters
arenaPointer to the arena.

◆ arena_pop_to()

void arena_pop_to ( Arena * arena,
usize pos )

Rolls the arena back to a specific byte offset.

All memory at or above pos is considered freed. The committed physical RAM is not returned to the OS — it stays mapped for future allocations.

Note
Prefer ArenaTemp / arena_temp_end() for scoped rollbacks.
Warning
In debug builds, passing a pos larger than the current arena position triggers an ASSERT, since that would silently be a no-op and almost certainly indicates a logic error in the caller.
Parameters
arenaPointer to the arena.
posTarget offset. Must be <= arena->pos in debug builds.

char * arena_push_cstr ( Arena * arena,
const char * str,
usize len )

Copies a string into the arena and null-terminates it.

The resulting buffer is len + 1 bytes and is always NUL-terminated, making it safe to pass directly to any C string API.

Example:

char *copy = arena_push_cstr(&arena, src, strlen(src));
