cbase

A high-performance, modern C project template designed to provide a clean, zero-dependency starting point for professional C development.

This template drags C into the modern era by providing "Quality of Life" features commonly found in languages like Odin, Zig, and Rust — such as string slices, exact-width types, and virtual memory arenas — without sacrificing the raw speed and simplicity of standard C.

Features

  • No Bloat: Zero external dependencies. Everything is built from scratch.
  • Virtual Memory Arenas: A professional Arena allocator built directly on top of OS-level virtual memory (VirtualAlloc / mmap). Reserve gigabytes of address space instantly and commit physical RAM only as needed. Say goodbye to malloc, free, and memory leaks.
  • Sized Strings (Slices): Fast, allocation-free UTF-8 string manipulation using pointer/length pairs (String8). Handles all of Unicode — completely eliminating buffer overflows and strlen overhead.
  • Modern Types & Macros: Exact-width integers (u8, i32, usize), type-safe evaluation macros (MIN, MAX, CLAMP), and intrusive linked-list tools.
  • Generic Allocator Interface: A lightweight vtable (Allocator) that lets library code stay allocator-agnostic — swap between an arena, the heap, or a custom pool at the call site without changing any internals.
  • Professional Logging: A thread-safe, leveled logger with colored console output and multi-sink (file/callback) support.
  • Built-in Testing: A lightweight, Unity-Build based testing harness that compiles and runs in milliseconds.
  • Frictionless Build System: A highly tuned CMake configuration that uses CONFIGURE_DEPENDS to automatically detect new .c files without needing manual CMake reloads. Supports both MSVC and Ninja generators out of the box.

Getting Started

Clone the Template

git clone https://gitlab.com/Alfred-Jijo/ccdb.git my_project
cd my_project

Build the Project

This project uses CMake (minimum v3.13) wrapped in convenient build scripts. You don't need to invoke CMake directly.

Windows

Command Result
build MSVC — Debug
build release MSVC — Release
build ninja Ninja + MSVC cl — Debug
build ninja release Ninja + MSVC cl — Release
build docs Generate Doxygen documentation
build clean Delete the build/ directory

Note: The ninja targets require cl.exe on your PATH. Run from a Developer Command Prompt or after calling vcvarsall.bat.

Linux / macOS

Command Result
./build.sh Default generator — Debug
./build.sh release Default generator — Release
./build.sh ninja Ninja — Debug
./build.sh ninja release Ninja — Release

Run the Application

Compiled binaries are placed in build/bin/.

  • Linux / macOS: ./build/bin/debug_cbase

Run the Tests

String Handling

String8 is a UTF-8 string slice — a pointer/length pair that covers all of Unicode without any heap allocation. It is not null-terminated, so it is safe to take substrings of arbitrary data without copying.

String8 path = STR8("src/main.c");
String8 dir = str8_prefix(path, 3); // "src"
String8 file = str8_skip(path, 4); // "main.c"
String8 ext = str8_suffix(path, 2); // ".c"
String8 noext = str8_chop(path, 2); // "src/main"
b32 is_c = str8_ends_with(path, STR8(".c")); // 1
usize sep = str8_find_first(path, '/'); // 3
// Safe to use directly in printf:
printf("%.*s\n", STR8_FMT(file));

Memory / Arena

// Reserve 1 GB of virtual address space (costs no physical RAM yet).
Arena arena = arena_create(GB(1));
// Allocate from the arena — zero-initialized, no free required.
MyStruct *s = PUSH_STRUCT(&arena, MyStruct);
u32 *buf = PUSH_ARRAY(&arena, u32, 1024);
float *v = PUSH_ARRAY_ALIGNED(&arena, float, 64, 32); // 32-byte SIMD alignment
// Scoped scratch allocation — rolls back automatically.
ArenaTemp tmp = arena_temp_begin(&arena);
u8 *scratch = PUSH_ARRAY(&arena, u8, 512);
arena_temp_end(tmp); // scratch is gone, permanent allocations survive
// Reset the arena for the next frame (committed RAM stays mapped).
arena_clear(&arena);
// Reset and return physical RAM to the OS (useful after large one-off loads).
arena_clear_and_decommit(&arena);
// Done — release the entire reservation.
arena_release(&arena);

Generic Allocator Interface

Library code that needs to allocate memory accepts an Allocator by pointer, making it independent of the concrete allocator:

Arena arena = arena_create(MB(64));
Allocator ally = arena_allocator(&arena);
// Or use the standard heap:
Allocator heap = gpalloc();
MyStruct *s = (MyStruct *)allocator_alloc(&ally, sizeof(MyStruct));
allocator_free(&ally, s, sizeof(MyStruct)); // no-op for arenas, works generically
gpalloc_destroy(&heap); // frees the internal context; your allocations are your responsibility
