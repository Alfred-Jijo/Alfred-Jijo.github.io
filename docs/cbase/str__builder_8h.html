Detailed Description

Arena-backed StringBuilder for efficient string concatenation.

A StringBuilder accumulates string segments as a singly-linked list of String8Node chunks, each allocated from a caller-supplied Arena. No copying occurs during appending — the final contiguous String8 is only materialized when sb_join() is called.

Design
Nodes (and their string data for formatted/cstr appends) are allocated from an arena supplied at each append call. Typically this would be a short-lived scratch arena that you clear after consuming the joined result:
Arena scratch = arena_create(MB(1));
Arena result_arena = arena_create(MB(4));
StringBuilder sb = {0};
sb_append(&sb, &scratch, STR8("Hello, "));
sb_append_fmt(&sb, &scratch, "%s!", "World");
String8 result = sb_join(&sb, &result_arena);
// result.str is in result_arena; scratch can be cleared freely.
arena_clear(&scratch);

Why caller decides the arena at join time
The builder nodes are cheap scratch allocations — they only need to live long enough to produce the final string. Passing a separate arena to sb_join() lets you put the result exactly where it needs to live (e.g., a permanent arena) while discarding the intermediate nodes cheaply.
