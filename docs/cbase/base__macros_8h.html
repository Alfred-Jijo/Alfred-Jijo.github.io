Detailed Description

Context detection, compiler abstractions, utility macros, and data structures.

A zero-dependency header that determines the compilation context (OS, compiler, architecture, endianness, C standard) and exposes the results as simple macros. Also provides the core QoL macro library used across the entire codebase.

Context Detection
At the top of the file, predefined compiler macros are inspected to set a canonical set of COMPILER_*, OS_*, ARCH_*, ENDIAN_*, and C_STANDARD macros. This means downstream code never has to repeat the brittle #if defined(_MSC_VER) chains - it just checks #if COMPILER_MSVC or #if OS_WINDOWS.
Macro Categories
  • Linkage (function, global, local_persist, external, thread_local)
    • Semantic replacements for C storage-class keywords.
  • Utility (MIN, MAX, CLAMP, SWAP, ARRAY_SIZE, STRINGIFY, GLUE)
    • Type-safe math and token manipulation. On GCC/Clang the typeof-based versions of MIN/MAX/CLAMP prevent double-evaluation bugs; MSVC gets safe fallbacks.
  • Alignment (ALIGN_UP_POW2, ALIGN_DOWN_POW2, IS_POW2, IS_POW2_OR_ZERO)
    • Fast bitwise alignment used by the arena allocator.
  • Memory Safety (MEM_ZERO, MEM_COPY, MEM_ZERO_STRUCT, etc.)
    • Wrappers around memset/memmove that enforce sizeof safety.
  • Intrusive Linked Lists (DLL_*, SLL_*)
    • Zero-allocation macro-based list operations. Nodes carry their own next/prev pointers rather than being wrapped in a generic container.
  • Debug (ASSERT, ASSERT_NOT_NULL, debug_break)
    • Assertions that compile out completely in release builds.
