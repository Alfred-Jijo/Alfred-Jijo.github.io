ccDB

This is a modern, modular C/C++ project template designed to provide a clean and organized starting point for your development. It features a modern CMake architecture, pre-configured Doxygen API generation, and high-quality baseline headers for type safety and compiler abstractions.

Features & Planned Additions

  • Core Headers
    • types.h: Strict, cross-platform fixed-width integer and float aliases (e.g., u8, i32, f32), and memory constants (MB(), GB()).
    • macros.h: Zero-dependency compilation context detection (OS, Compiler, Arch), type-safe math macros (MIN, MAX, CLAMP), and standard debugging assertions.
  • Planned Subsystems
    • Logging (Basic stdout/stderr support included via macros)
    • Allocators
      • Arena
      • Bump
      • Pool
    • Data Structures and Algorithms
      • Linked List
      • Array List
      • Stacks & Queues

How to Build

This project uses modern, target-based CMake and provides helper scripts (build.bat for Windows, build.sh for Linux/macOS) to make compiling effortless.

Debug Build (Default)

Generates an unoptimized executable with debug symbols and enables the DEBUG macro in your C code.

  • Windows: Run build
  • Linux/Mac: Run ./build.sh
  • Output: build/bin/debug_ccdb

Release Build

Generates a highly-optimized executable with no debug symbols and enables the RELEASE macro in your C code.

  • Windows: Run build release
  • Linux/Mac: Run ./build.sh release
  • Output: build/bin/release_ccdb

Generating API Documentation

This project comes pre-configured with Doxygen to generate a searchable, beautifully formatted HTML website of your codebase (including call graphs and caller graphs). Ensure you have Doxygen and Graphviz installed on your system.

  • Windows: Run build docs
  • Linux/Mac: Run ./build.sh docs

Once complete, open docs/html/index.html in your web browser to view the API reference.

Adding Dependencies

To add third-party libraries (like GLFW, Raylib, etc.):

  • Drop their source code folders into the lib/ directory.
  • Add add_subdirectory(library_name) to lib/CMakeLists.txt.
  • Add the library name to the target_link_libraries list in src/CMakeLists.txt.
  • Run your build script. CMake will automatically compile them as static libraries and link them to your main executable.
