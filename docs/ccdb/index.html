ccDB

This is a modern, modular C/C++ project template designed to provide a clean and organized starting point for your development. It features a modern CMake architecture, pre-configured Doxygen API generation, and high-quality baseline headers for type safety and compiler abstractions.

Features & Subsystems

  • Core Headers
  • types.h: Strict, cross-platform fixed-width integer and float aliases (e.g., u8, i32, f32), and memory constants (MB(), GB()).
  • macros.h: Zero-dependency compilation context detection (OS, Compiler, Arch), type-safe math macros (MIN, MAX, CLAMP), and standard debugging assertions.
  • Warden Subsystem
  • [x] OS Abstraction & Wrappers
  • [x] Utilities
  • [x] Arena Allocator
  • [ ] Bump Allocator
  • Data Structures
  • [ ] Linked List
  • [ ] Array List
  • [ ] Stacks & Queues

How to Use This Template

Clone the Repository

Get this template to your local machine:

git clone https://gitlab.com/Alfred-Jijo/ccdb.git my_project
cd my_project

Building the Project (CMake)

This project uses modern, target-based CMake. It separates the build artifacts from your source code to keep the repository clean.

# Create a build directory and configure the project
mkdir build
cd build
cmake ..
# Compile the project
cmake --build .

Your compiled executable will be placed in the build/bin/ directory.

Adding Dependencies

To add third-party libraries (like GLFW, Raylib, etc.), drop their source folders into the lib/ directory, add them to lib/CMakeLists.txt, and CMake will automatically compile them as static libraries and link them to your main executable.

Generating API Documentation

This project comes pre-configured with Doxygen to generate a searchable, beautifully formatted HTML website of your codebase (including call graphs and caller graphs).

Ensure you have Doxygen and Graphviz (for diagrams) installed on your system.

# From inside your build folder:
cmake --build . --target docs

Once complete, open docs/html/index.html in your web browser.
